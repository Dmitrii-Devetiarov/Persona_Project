# Latent Persona Memory

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Python 3.11](https://img.shields.io/badge/python-3.11-blue.svg)](https://www.python.org/downloads/release/python-3110/)
[![Docker](https://img.shields.io/badge/Docker-Ready-2496ED?logo=docker&logoColor=white)](https://www.docker.com/)

> Надстройка над LLM для имитации персоны модели, позволяющая переносить стиль общения и контекст между диалогами. Стиль персоны формируется не фактологически, а по манере изложения пользователя (краткость, эмоциональность, сухость и прочее). Память нечёткая, выделяет из предшествующих диалогов только краткий контекст.

## ✨ Возможности

- **Потоковый анализ стиля:** Автоматическое определение 20 лингвистических признаков текста
- **Семантическая память:** Сохранение контекста диалогов без дословного запоминания фактов
- **Адаптивное поведение:** Модель подстраивается под манеру общения по 8 независимым осям
- **Полная контейнеризация:** Запуск в один клик через Docker, все зависимости включены
- **Гибкое управление:** Ручная настройка всех параметров персоны через UI с возможностью добавления пользовательских инструкций как раздела системного промпта
- **Билингвальность:** Полная поддержка русского и английского языков (интерфейс, детекция маркеров, промпты)

## 🏗️ Архитектура

Проект построен на четырёх параллельных потоках анализа, которые сходятся в ядре персоны (Persona Core) для генерации итогового промпта для LLM. Все компоненты билингвальны — детекция стоп-слов, маркеров стиля и промптов работает на русском и английском языках.

| Поток | Назначение | Метод |
|-------|------------|-------|
| **A — Fuzzy Memory** | Семантическое облако диалогов | E5-large + KeyBERT + ассоциативная память |
| **B — Surface Style** | Лингвистические признаки текста | spaCy + pymorphy3 (20 признаков) |
| **C1 — Explicit** | Прямые инструкции пользователя | Prototype embeddings, 8 осей |
| **C2 — Implicit** | Реакции на ответы модели | Анализ follow-up сообщений |
| **Fusion** | Объединение потоков | Адаптивные веса, 8 осей |

### Оси персоны

| Ось | Диапазон | Описание |
|-----|----------|----------|
| `emotionality` | –1 (сухо) ↔ +1 (эмпатия) | Уровень эмоциональной вовлечённости |
| `factual_accuracy` | –1 (максимально широкий кругозор) ↔ +1 (глубокая проработка наиболее принятой версии) | Широта версий и глубина проработки |
| `verbosity` | –1 (кратко) ↔ +1 (детально) | Длина и подробность ответов |
| `figurativeness` | –1 (простая речь) ↔ +1 (образная, литературная) | Насыщенность и сложность изобразительного языка |
| `disagreement` | –1 (согласие) ↔ +1 (дискуссия) | Склонность к возражениям |
| `comfort` | –1 (панибратство) ↔ +1 (формальная вежливость) | Уровень вежливости и дистанции |
| `model_resistance` | 0 (уступать давлению пользователя) ↔ +1 (держать линию поведения) | Устойчивость модели к давлению пользователя |
| `complexity` | –1 (просто) ↔ +1 (терминологично) | Терминологическая сложность языка |

## 🚀 Быстрый старт

# Быстрый старт

## Необходимые условия

- [Docker](https://docs.docker.com/get-docker/), установленный на вашем компьютере
- API-ключ Yandex Cloud с доступом к DeepSeek

## Вариант 1: Загрузка из GitHub Container Registry

```bash
# Загрузите последний образ
docker pull ghcr.io/dmitrii-devetiarov/persona-project:latest

# Создайте директорию проекта и перейдите в неё
mkdir persona-memory && cd persona-memory

# Настройте переменные окружения
cp .env.example .env
# Отредактируйте .env, указав свои YANDEX_API_KEY и YANDEX_FOLDER_ID

# Создайте пустую директорию data для хранения персоны
mkdir -p data

# Запустите приложение
docker compose up
```
Откройте http://localhost:8501 в браузере.

## Вариант 2: Сборка из исходников

```bash
# Клонируйте репозиторий
git clone https://github.com/Dmitrii-Devetiarov/Persona_Project.git
cd Persona_Project

# Настройте переменные окружения
cp .env.example .env
# Отредактируйте .env, указав свои YANDEX_API_KEY и YANDEX_FOLDER_ID

# Соберите и запустите
docker compose up
```
Откройте http://localhost:8501 в браузере.
